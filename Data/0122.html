Just to expand on the idea of transformations:

My working definition of a "grid" is "an array such that values next to
each other in index space are next to each other in coordinate space ".
 (I think I also need the requirement that the number of dimensions
equals the number of coordinate axes) An invertible, continuous
transformation will preserve this "next to each other" property. Im
sure someone who remembers more toplogy than i do could say this more
precisely.

In the NcML Coordinate System spec we are interested in Coordinate
Transforms that have this property. If anyone envisions using CF for
other kinds of coordinate systems, i would be interested in knowing
about it.
